## 페이지 구성

`documents/landing_page_10_core_strategies.md`의 전략과 원본 docx(`-7월_1일(수)_...무료_특강-20260622_111738.docx`) 내용을 반영해 구성.

1. **히어로** — 공감 스토리 헤드라인("같은 폰, 같은 풍경인데 왜 그 사람 사진만 작품이 될까요?") + 일정/선착순 배지 + 통계(2시간/100명/100%) + 인트로 영상
2. **공감/고민 섹션** — 원본 콘텐츠의 의문문("왜 내 사진은 어둡고 칙칙할까" 등)을 카드 6개로 구성
3. **핵심 전환 문장** — "비싼 장비도, 복잡한 이론도 아닙니다 — 시선과 보정의 감각입니다"
4. **무료 특강 안내** — 강의명/일시/방식/강사/신청방법 정리
5. **커리큘럼 4단계** — 촬영 기본기 → Snapseed 보정 → AI 확장 콘텐츠 → 사진강사로 도약
6. **이 강의를 들어야 하는 이유** 5가지
7. **강사 소개** — 엘란비탈 박성욱 작가 (images/director.jpg 재사용)
8. **CTA** — 선착순 100명 스카시티 강조

디자인은 기존 `lecture-landing.html`(AI 에이전트 특강) 구조를 베이스로 했고, 색상만 보라/파랑 → 호박색/레드 그라디언트로 변경해 사진 강의 톤에 맞춤.

## 신청 폼 처리 흐름

처음엔 기존 공용 Apps Script(`google-apps-script.gs`, 후기·AI특강·회원·문의 처리 겸용)에 분기를 추가해 사진특강 신청도 같이 처리하려 했으나:

- 그 공용 스크립트의 실제 배포 위치를 찾기 어려웠고 (여러 "제목 없는 프로젝트" standalone 스크립트 후보 중 식별 필요)
- 사용자가 애초에 원했던 건 신청 데이터를 별도의 **공식 신청서 스프레드시트**(`2026 0701 스마트폰 사진특강 신청서(응답)`, 구글폼 forms.gle/kKJSmXqoijhYW83f6 응답 시트)에 모으는 것이었음

→ **방향 전환**: 공용 스크립트는 원상 복구하고, 이 페이지 전용의 독립 Apps Script(`photo-lecture-apps-script.gs`)를 새로 만들어 신청서 스프레드시트에 직접 바인딩.

### photo-lecture-apps-script.gs가 하는 일 (doGet)

1. 이름/이메일/연락처/동의 여부를 받아 응답 시트(gid=12733408)에 한 행 추가
   - 열 순서: 타임스탬프, 점수, 이름, **신청 경로("랜딩페이지"로 고정)**, 전화번호, 이메일, 배우고 싶은 것, 기대하는 것, 녹화본 구입, **개인정보 동의**, 저작권 동의
   - 랜딩페이지에서 받지 않는 항목(점수·배우고 싶은 것 등)은 빈 칸으로 둠
   - 개인정보 동의는 폼의 실제 체크박스 응답을 그대로 반영 (가짜로 "동의함" 채우지 않음)
2. 신청자에게 확인 메일 발송 (매 신청마다 즉시 발송)
3. 관리자(elanvital7@naver.com)에게 신청 알림 메일 발송 — **누적 10명 단위 배치**로 발송 (아래 "관리자 알림 정책" 참고)

## 관리자 알림 정책 (신청 10명마다 배치 알림)

신청이 들어올 때마다 관리자에게 메일이 가면 너무 자주 와서 정신없다는 피드백에 따라, **관리자 알림은 누적 신청자 수가 10의 배수가 될 때만(10명, 20명, 30명...) 발송**하도록 바꿈. 이 정책은 사진특강·코딩 수업(AI 에이전트 특강) 양쪽 모두에 동일하게 적용:

- `photo-lecture-apps-script.gs`: `_sendAdminNotify` → `_sendAdminDigest`로 교체. `total % 10 === 0`일 때만 호출.
- `google-apps-script.gs`: `_sendLectureAdminNotify` → `_sendLectureAdminDigest`로 교체. 동일하게 `total % 10 === 0`일 때만 호출.
- 매번 단순히 건너뛰는 게 아니라, 호출될 때 **최근 10명의 신청일시·이름·이메일·연락처를 표로 정리**해서 한 번에 보여줌 (시트에서 최근 10행을 읽어옴).
- 신청자 본인에게 가는 확인 메일은 영향 없음 — 그건 계속 매 신청마다 즉시 발송.
- 코드 수정 후 두 스크립트 모두 재배포 완료 (사진특강 전용 스크립트 + 공용 스크립트 둘 다).

### 배포 정보

- 스프레드시트: `https://docs.google.com/spreadsheets/d/1mWvHb6H6YhtAe-j2y_PDGwfcK4jY8mpHyCO0OJHBA-0/edit?gid=12733408`
- 이 스프레드시트에 바인딩된 Apps Script에서 직접 배포 (확장 프로그램 → Apps Script → 배포 → 새 배포 → 웹 앱)
- 배포된 웹 앱 URL을 `photo-lecture-landing.html`의 `SCRIPT_URL`에 연결 (현재 연결됨)
- 코드 수정 시 재배포 필요: 배포 → 배포 관리 → 기존 배포 연필 아이콘 → 새 버전 → 배포 (URL은 유지됨)

## 신청 폼 필드 규칙 (통일)

`photo-lecture-landing.html`과 `lecture-landing.html`(AI 에이전트 특강) 모두 동일하게 적용:

- 이름, 이메일: 필수
- **연락처: 필수**, `010-1234-5678` 형식 검증 (정규식 `^01[0-9]-\d{3,4}-\d{4}$`)
- **개인정보 수집·이용 동의 체크박스: 필수** (미체크 시 제출 불가)
- 하단 안내문: "*제출 버튼을 클릭하시면 개인정보 수집 및 이용에 동의한 것으로 간주됩니다."
